# ETF Review OS — Dutch terminology preferences

Use this file to keep the Dutch companion report natural, premium, and consistent.

## Core rule
The Dutch report should read like high-quality Dutch written for serious subscribers.
It must preserve the exact substance of the canonical English report, but should avoid awkward literal translations.

## Preferred translations
- sleeve -> positie, allocatie, rol, or bouwsteen depending on context
- resilience sleeve -> beschermende positie or weerbaarheidspositie
- lane -> thema
- carry-forward -> doorgeschoven waardering
- funded switch -> rotatie or portefeuillerotatie
- resilience -> weerbaarheid or veerkracht
- actionable now -> nu inzetbaar
- scale in slowly -> rustig opbouwen
- watchlist -> watchlist
- too early -> te vroeg
- pullback -> terugval
- breadth -> breedte
- leadership -> leiderschap
- holdings -> posities
- core beta -> kernblootstelling brede markt

## Avoid awkward phrasing
- weerbaarheidsbouwstenen
- gefinancierde wissel

## Compatibility reminders
- Keep tickers exact.
- Keep numbers exact.
- Prefer natural Dutch over Anglo-Dutch hybrids.
- Keep the title `Weekly ETF Pro Review` in English unless explicitly changed later.
- Keep table and section structure aligned with the English canonical report.
- Keep Section 2 bucket labels in English when needed for rendering compatibility: `Add`, `Hold`, `Hold but replaceable`, `Reduce`, `Close`.

## Section 9 discipline
The Second-Order Effects Map must contain genuine macro, market, policy, geopolitical, sector, earnings, cost, or positioning effects.
Use the Data note or valuation note for pricing-pass and verification-pass wording.
